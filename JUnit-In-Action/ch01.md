# 1장 JUnit 첫걸음

소프트웨어 개발의 역사에서 이토록 많은 사람이 이렇게 짧은 코드로부터 이토록 큰 도움을 받은 적이 없었다. - 마틴 파울러(Martin Fowler)

```java

package Junit_In_Action.examples;

/**
 * 계산기 클래스
 * double 형 파라미터 두개를 입력받아 그 합을 double형으로 반환하도록 설계되었다.
 * 문법 오류가 없는 것은 바로 알 수 있지만, 런타임에도 잘 작동하는지는 알 길이 없다.
 * 단위 테스트의 모토는 '프로그램에서 자동화된 테스트를 거치지 않은 기능은 존재하지 않는다' 이다.
 * 정말로 이 기능이 잘 동작하는지 검증하기 위한 자동화된 테스트를 만들어야 한다.
 */
public class Calculator {

    public double add(double number1 , double number2){
        return number1 + number2;
    }
}

/**
 * add 메서드는 너무 단순해서 문제가 날게 없지 않나?
 * 그렇다. 하지만 계산기 프로그램에서는 어엿한 하나의 기능이다. 프로그램에 대한 확신을 가지기 위해서는
 * add 를 검증하는 테스트를 대부분의 개발자는 기대할 것이다. 구현이 아무리 단순하더라도 말이다.
 *
 */
```

```java
package Junit_In_Action.examples;


/**
 * 정해진 값들을 메서드에 입력하고 그 반환 값이 예상 값과 일치하는지 확인하는 테스트 프로그램.
 */
public class CalculatorTest {
    public static void main(String[] args){
        Calculator calculator = new Calculator();
        double result = calculator.add(10, 50);
        if(result != 60){
            System.out.println("Bad result: "+ result);

        }
    }
}

/**
 * 예상 값과 다르다면 메시지를 출력한다.
 * 하지만, 예상 값과 일치하면 아무런 메시지 없이 성공할 것이다.
 * 만약, 코드를 수정해서 테스트가 실패하면 프로그램이 무엇을 출력하는지 직접 살펴야 한다.
 * 테스트 대상은 코드이지, 우리 자신이 아니다.
 */
```
